Request and view events that have taken place within a specific timeframe.

With the Events API, you can:

- [Get an event based on an ID](/reporting-and-insights/events-api#EventsAPI-getevent)
- [Get a summary of notifications for an event](/reporting-and-insights/events-api#EventsAPI-getnotificationsummary)
- [Retry notifications for all subscribed active webhooks](/reporting-and-insights/events-api#EventsAPI-retrynotifications)
- [Retry a single event for a specified webhook](/reporting-and-insights/events-api#EventsAPI-retrysingleevent)

To view a list of events you can subscribe to, please see our [event types page](/reporting-and-insights/events-api/event-types).

---

### Retry notifications for all active webhooks

Use this API request to retry notifications for all subscribed active webhooks.

Use the details below to set up your request.

#### Endpoints

You can find the full list, as well as complete request and response examples, in our <ExternalLink href="https://api-reference.checkout.com" target="_blank">API reference</ExternalLink>.

<tabs>
<tab title="Live">
<LineCodeWrapper type="post">

`https://api.checkout.com/events/{eventId}/webhooks/retry`

</LineCodeWrapper>
</tab>
<tab title="Sandbox">
<LineCodeWrapper type="post">

`https://api.sandbox.checkout.com/events/{eventId}/webhooks/retry`

</LineCodeWrapper>
</tab>
</tabs>

#### Response

Below is a list of example responses:

- 202 - Accepted. Notifications were queued
- 204 - No content. No active webhooks are subscribed to this event
- 404 - The event does not exist
